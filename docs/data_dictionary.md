# Data Dictionary
## Overview

This document describes every data element used by gd-Pipeline.

Most fields originate from Geode mods (Death Tracker and Playtime Tracker). Others, such as `completed` and `completion_date`, are generated by gd-Pipeline during data processing.

Each entry documents the field's meaning, origin, data type and relevant implementation details.

## canonical_id

| Property | Value |
|----------|-------|
| Source | Death Tracker |
| Type | TEXT |
| Nullable | No |

### Description

Unique identifier used internally by gd-Pipeline (Used as the primary key).

It uniquely identifies every tracked level variant and is derived from the Death Tracker filename.

### Examples

```
144807542
144807542-daily
144807542-gauntlet
13-editor
5-local
```

## level_id

| Property | Value |
|----------|-------|
| Source | Death Tracker |
| Type | TEXT |
| Nullable | No |

### Description

Original Geometry Dash level identifier.

Unlike `canonical_id`, this value represents the original identifier assigned by the game and does not distinguish between level variants such as Daily or Gauntlet.

### Example

```
144807542
13-editor
5-local
```

## difficulty

| Property | Value |
|----------|-------|
| Source | Death Tracker |
| Type | INTEGER |
| Nullable | No |

### Description

Difficulty value assigned by Death Tracker.

gd-Pipeline preserves this value without modification.

### Encoding

| Value | Difficulty |
|------:|------------|
| -1 | Auto |
| 0 | N/A |
| 1 | Easy |
| 2 | Normal |
| 3 | Hard |
| 4 | Harder |
| 5 | Insane |
| 6 | Easy Demon |
| 7 | Medium Demon |
| 8 | Hard Demon |
| 9 | Insane Demon |
| 10 | Extreme Demon |

### Notes

This encoding is defined by Death Tracker and is not interpreted by gd-Pipeline.

## attempts

| Property | Value |
|----------|-------|
| Source | Geometry Dash |
| Type | INTEGER |
| Nullable | No |

### Description

Official attempt count reported by Geometry Dash.
This value is extracted from the Geometry Dash metadata and includes every attempt recognized by the game.

If linked levels are configured in Death Tracker, attempts from every linked level are included in the total.

### Example

```
4235
30065
25
```
### Notes

gd-Pipeline will **NOT** link levels automatically, anyone using this program will have to link levels manually.

## tracked_attempts

| Property | Value |
|----------|-------|
| Source | Death Tracker |
| Type | INTEGER |
| Nullable | No |

### Description

Attempt count reported by Death Tracker.

Unlike the official Geometry Dash attempt counter, this value follows Death Tracker's own tracking rules.

### Example
```
4235
30065
25
```

### Notes

The value may differ from `attempts` because Geometry Dash and Death Tracker do not count attempts using the same criteria.

gd-Pipeline stores both values exactly as reported by their respective sources and does not attempt to reconcile the difference.

## current_best

| Property | Value |
|----------|-------|
| Source | Death Tracker |
| Type | INTEGER |
| Nullable | No |

### Description

Highest completion percentage achieved on the level.

The value ranges from 0 to 100. A value of 100 indicates that the level has been completed.

### Example

```
94
50
```

## worst_fail

| Property | Value |
|----------|-------|
| Source | Death Tracker |
| Type | INTEGER |
| Nullable | No |

### Description

Highest completion percentage reached before the successful completion attempt.

The value ranges from 0 to 99.

### Example

```
97
```

### Notes

This value remains unchanged after the level has been completed.

## playtime

| Property | Value |
|----------|-------|
| Source | Playtime Tracker |
| Type | INTEGER |
| Nullable | No |
| Unit | Seconds |

### Description

Total time spent playing the level.

The value is calculated by summing the duration of every recorded session provided by Playtime Tracker.

### Example

```
64872
```

### Notes

Although Geometry Dash players commonly express playtime in hours, gd-Pipeline stores the raw duration in seconds to preserve precision.

## completed

| Property | Value |
|----------|-------|
| Source | gd-Pipeline |
| Type | BOOLEAN |
| Nullable | No |

### Description

Indicates whether the player has completed the level.

Once this value becomes `TRUE`, the pipeline no longer updates the record.

## completion_date

| Property | Value |
|----------|-------|
| Source | gd-Pipeline |
| Type | DATE |
| Nullable | Yes |

### Description

Date on which the player completed the level for the first time.

The value remains `NULL` until the level is completed.


### Example

```
2015-11-03
```
